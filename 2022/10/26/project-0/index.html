安全检测工具(DEMO) 说明文档

1.程序运行环境

操作系统：Ubuntu 22.04
Python 解释器：Python 3.10
Python 运行库：os与tkinter

2.程序简介

2.1 功能支持

目前支持的安全编译选项测试有：Fortify,PIE(ASLR),PIC,Rpath,Stack Canary
trapv,RELRO和NX(noexecutable)
目前尚不支持检查参数

2.2 源码简介

2.2.1 特征提取部分

特征提取分三个部分：文件符号表的提取、特征的检测、结果的显示

2.2.1.1 文件符号表提取（以获取.dynamic字段为例）

dynamic_segment_get(){`
log_2="$(readelf -d "$1")"`
arr_2=($(echo $log_2 | tr " " "\n")) `
echo "${arr_2[@]}" > dynamic_segement.txt`
}

2.2.1.2 特征的检测（以fortify为例）

FORTIFY(){
compile_code_get "$1"
local fea="<__strcpy_chk@plt>"
for i in "${arr_3[@]}"
do
if [ "$fea" == "$i" ];
then 
flag_fortify=1
break
fi
done
}

2.2.1.3 显示部分(以fortify为例)

if [ $1 -eq 1 ] && [ $flag_fortify -eq 1 ] 
then
echo "Fortify: ON"
fi
if [ $1 -eq 1 ] && [ $flag_fortify -eq 0 ]
then
echo "Fortify: OFF"
fi

2.2.2 显示界面部分

使用的是tkinter内置库，设置和选项选择，结果显示，文档选择三个界面

3.使用方法

3.1 方法1：直接使用封装后的.exe文件

点击📂dist文件夹中的PRP41_demo文件，直接进入界面

3.2 方法2：使用python运行.py源码

在文件中打开terminal运行python3 PRP41_demo

3.3 其他

如果收到Permission denied的情况，可在terminal中尝试运行chmod +x PRP41_demo main_v4
由于本程序的错误处理部分尚未完成，请选择.so文件进行检测，在文件夹中有test作为测试样例
